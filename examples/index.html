Choicey – User Documentation

Choicey is a flexible, searchable multiselect input. This guide explains all options and shows practical examples for using it in your forms.

1. Basic Usage

Create a multiselect with predefined countries:


        <div data-controller="choicey" data-choicey-items-value='[
            { "value": "PL", "text": "Poland 🇵🇱", "meta": "Some meta information" },
            { "value": "CN", "text": "China 🇨🇳" },
            { "value": "BR", "text": "Brazil 🇧🇷" },
            { "value": "DE", "text": "Germany 🇩🇪" },
            { "value": "CA", "text": "Canada 🇨🇦" },
            { "value": "AU", "text": "Australia 🇦🇺" },
            { "value": "IN", "text": "India 🇮🇳" },
            { "value": "FR", "text": "France 🇫🇷" },
            { "value": "ZA", "text": "South Africa 🇿🇦" },
            { "value": "JP", "text": "Japan 🇯🇵" },
            { "value": "MX", "text": "Mexico 🇲🇽" }
          ]'
          data-choicey-search-placeholder-value="Search countries..."
          data-choicey-clear-all-value="true"
        >
          <select multiple class="choicey__hidden" data-choicey-target="hidden" name="countries[]"></select>
        </div>

Size variants:

  • choicey__sm –x compact version
  • choicey__md – medium (default)

2. Single value

Create a single-select input:


          <div data-controller="choicey" data-choicey-preload-url-value="/api/cars" data-choicey-single-value="true" data-choicey-search-placeholder-value="Search cars...">
            <select class="choicey__hidden" data-choicey-target="hidden" name="cars[]"></select>
          </div>

3. Selected items limit


          <div data-controller="choicey" class="choicey__sm"
            data-choicey-preload-url-value="http://localhost:3001/elements"
            data-choicey-search-url-value="http://localhost:3001/elements"
            data-choicey-selected-limit-value="3"
            data-choicey-limit-help-text-value="{n} / {limit} 已选"
            data-choicey-limit-reached-text-value="(已达上限)"
            data-choicey-search-placeholder-value="Search cars..."
          >
            <select class="choicey__hidden" data-choicey-target="hidden" name="cars[]"></select>
          </div>

4. Showing only count

            <select class="choicey__hidden" data-choicey-target="hidden" name="cars[]"></select>

5. Preloading Items from API

You can load options from a JSON endpoint when the component loads:


          <div data-controller="choicey" data-choicey-preload-url-value="/api/cars" data-choicey-search-placeholder-value="Search cars...">
            <select multiple class="choicey__hidden" data-choicey-target="hidden" name="cars[]"></select>
          </div>

6. Remote Search

Search items via API dynamically as the user types:


          <div data-controller="choicey" data-choicey-search-url-value="/api/cars/search" data-choicey-search-placeholder-value="Search cars...">
            <select multiple class="choicey__hidden" data-choicey-target="hidden" name="cars[]"></select>
          </div>
Note: For remote search, the API should accept a query parameter (default: q) and return a JSON array of items in the format: [{ "value": "1", "text": "Option 1" }, ...]. Another query param sent while searching is the preselects param that contains a set of already selected values in the multiselect (a string separated by a comma ",").

7. Preload + Remote Search

Initial data is loaded from the server and further searches are performed remotely.


          <div data-controller="choicey" data-choicey-preload-url-value="/api/cars" data-choicey-search-url-value="/api/cars/search" data-choicey-search-placeholder-value="Search cars...">
            <select multiple class="choicey__hidden" data-choicey-target="hidden" name="cars[]"></select>
          </div>

10. Adding New Items

Allow users to add custom items not in the list:


          <div data-controller="choicey" data-choicey-addable-url-value="/api/tags" data-choicey-addable-button-text-value="Add new tag"data-choicey-search-placeholder-value="Search or add tags...">
            <select multiple class="choicey__hidden" data-choicey-target="hidden" name="tags[]"></select>
          </div>
Available Addable Parameters:
  • data-choicey-addable-url-value: String – The endpoint to POST new items to.
  • data-choicey-addable-params-value: Object – Additional parameters to send with the addable request (as JSON object).
  • data-choicey-addable-min-chars-value: Number – Minimum characters required in the search box before the add button appears (default: 3).
  • data-choicey-addable-button-text-value: String – Text shown for the add button when no results are found. Default is Add ${query}

When a user adds a new item, a POST request is sent to addable-url with the following JSON body:

{
  addable: "user input text",
  ...addableParams
}

The server should return the new item as a JSON object: { "value": "...", "text": "..." }.

11. Customizing Item Appearance

You can add custom classes to items for badges or colors:


          {
            "value": "macaw",
            "text": "Macaw 🦜",
            "spanClass": "badge bg-success"
          }

12. Customizing Items with Classes

You can add custom classes to each element of an item:

  • liClass – class applied to <li>
  • labelClass – class applied to <label>
  • inputClass – class applied to <input type="checkbox">
  • spanClass – class applied to the <span> containing text
Example: custom colors, disabled input, badges

          {
            "value": "rooster",
            "text": "Rooster 🐓",
            "liClass": "bg-secondary p-2 mb-1 rounded",
            "labelClass": "d-flex align-items-center",
            "inputClass": "form-check-input me-2",
            "spanClass": "badge bg-danger",
            "disabled": true
          }

Effect:

  • The <li> gets custom background and padding
  • The <label> is flex-aligned
  • The checkbox input can be styled (e.g., Bootstrap form-check-input)
  • The text is wrapped in a colored badge
  • You can mark item as disabled using "disabled": true – the checkbox cannot be selected

13. Accessing Selected Values

After selection, the hidden select exposes a values property:


          const select = document.querySelector('[data-choicey-target="hidden"]')
          console.log(select.values) // ["cuckoo","macaw"]

14. Keyboard Navigation

  • ArrowDown – move focus to next item
  • ArrowUp – move focus to previous item
  • Enter – select focused item
  • Backspace – remove last selected item
  • Escape – clear search / close dropdown

15. Events

Listen for these events to react to user actions:

Event Detail
choicey-change Fired whenever selection changes
choicey-added Fired when an item is selected. event.detail.item contains the item object.
choicey-removed Fired when an item is removed. event.detail.id contains the item value.
choicey-addable Fired when the user clicks the addable button. 

          document.addEventListener('choicey-added', (event) => {
            console.log('Added:', event.detail.item)
          })

17. Grouped Items (Optgroup)

You can group items under headers by passing an array of group objects, each with a group label and an items array:


          <div data-controller="choicey" data-choicey-items-value='[
              { "group": "Fruits", "groupLabelClass": "bg-info", "items": [
                { "value": "apple", "text": "Apple 🍎" },
                { "value": "banana", "text": "Banana 🍌" },
                { "value": "orange", "text": "Orange 🍊" }
              ] },
              { "group": "Vegetables", "items": [
                { "value": "carrot", "text": "Carrot 🥕" },
                { "value": "broccoli", "text": "Broccoli 🥦" },
                { "value": "pepper", "text": "Pepper 🌶️" }
              ] }
            ]' 
            data-choicey-search-placeholder-value="Select food..."
            data-choicey-clear-all-value="true"
          >
            <select multiple class="choicey__hidden" data-choicey-target="hidden" name="food[]"></select>
          </div>
  • Each group object must have a group label and an items array.
  • All features (search, addable, clear-all, etc.) work with grouped data.
  • You can add a custom class to the group label using groupLabelClass.
